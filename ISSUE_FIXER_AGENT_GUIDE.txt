================================================================================
                    ISSUE FIXER AGENT - USER GUIDE
================================================================================

OVERVIEW
================================================================================

The Issue Fixer Agent is an advanced automated tool that scans your codebase
for various issues and automatically fixes many of them. It's more comprehensive
than the basic Error Fixer Agent, detecting and fixing:

- Console statements
- Missing error handling
- Security issues
- Performance problems
- Accessibility issues
- Code quality issues
- And more...

USAGE
================================================================================

Run the Issue Fixer Agent:
---------------------------

npm run fix-issues

Or directly:
  node issue-fixer-agent.js

WHAT IT DETECTS & FIXES
================================================================================

AUTOMATIC FIXES:
----------------

1. Console Statements
   - Detects: console.log(), console.error(), console.warn(), console.debug()
   - Action: Comments out console statements (keeps service worker logs)
   - Priority: Low
   - Files: All .js files

2. ESLint Auto-Fix
   - Runs ESLint with --fix flag
   - Fixes formatting and style issues
   - Priority: Low

ISSUES DETECTED (Manual Review Required):
------------------------------------------

3. Missing Error Handling
   - Detects: Async functions without try-catch blocks
   - Priority: Medium
   - Action: Manual review recommended

4. Security Issues
   - Detects: Hardcoded secrets, dangerous eval() usage
   - Priority: High
   - Action: Immediate review required

5. Missing Type Checking
   - Detects: Components without PropTypes or TypeScript
   - Priority: Low
   - Action: Consider adding PropTypes

6. Missing Key Props
   - Detects: Array.map() without key prop
   - Priority: Medium
   - Action: Add key prop to map functions

7. Performance Issues
   - Detects: Expensive operations without useMemo
   - Priority: Low
   - Action: Consider optimization

8. Accessibility Issues
   - Detects: Images without alt, buttons without aria-label
   - Priority: Medium
   - Action: Add accessibility attributes

9. Code Duplication
   - Detects: Repeated code patterns
   - Priority: Low
   - Action: Consider refactoring

10. useEffect Dependencies
    - Detects: useEffect with empty dependencies
    - Priority: Low
    - Action: Verify if intentional

ISSUE PRIORITIES
================================================================================

HIGH PRIORITY (Immediate Action):
----------------------------------
- Security vulnerabilities
- Hardcoded secrets
- Dangerous code patterns

MEDIUM PRIORITY (Review Soon):
-------------------------------
- Missing error handling
- Accessibility issues
- Missing key props

LOW PRIORITY (Consider Later):
-------------------------------
- Console statements
- Missing PropTypes
- Performance optimizations
- Code duplication

INTEGRATION
================================================================================

Add to package.json scripts:
  "precommit": "npm run fix-issues",
  "prebuild": "npm run fix-issues"

This will automatically check and fix issues before committing/building.

WORKFLOW INTEGRATION
================================================================================

Recommended Workflow:
---------------------

1. Before committing:
   npm run fix-issues

2. Review the report:
   - Check HIGH priority issues first
   - Review MEDIUM priority issues
   - LOW priority can be addressed later

3. Commit changes:
   git add .
   git commit -m "Auto-fix: Applied issue fixes"

4. Manual fixes:
   - Address HIGH priority issues
   - Fix MEDIUM priority issues
   - Consider LOW priority improvements

COMPARISON WITH ERROR FIXER AGENT
================================================================================

Error Fixer Agent:
- Basic syntax fixes
- Missing imports
- Simple code corrections
- Quick fixes

Issue Fixer Agent:
- Comprehensive issue detection
- Security scanning
- Performance analysis
- Accessibility checks
- Code quality metrics
- Detailed reporting

Use Error Fixer Agent for: Quick syntax fixes
Use Issue Fixer Agent for: Comprehensive code review

TROUBLESHOOTING
================================================================================

Issue: Agent reports false positives
Solution:
  - Review HIGH priority issues manually
  - Some patterns (like "password" in forms) are false positives
  - Adjust detection patterns in agent code

Issue: Agent comments out needed console statements
Solution:
  - Review commented code
  - Uncomment if needed
  - Use proper logging service instead

Issue: Too many issues reported
Solution:
  - Focus on HIGH priority first
  - Address MEDIUM priority gradually
  - LOW priority can wait

Issue: Agent doesn't fix certain issues
Solution:
  - Some issues require manual fixes
  - Review the issue type and message
  - Follow recommendations in report

BEST PRACTICES
================================================================================

1. Run regularly:
   - Before each commit
   - Before deployment
   - Weekly code reviews

2. Review reports:
   - Always check HIGH priority issues
   - Review MEDIUM priority issues
   - Track LOW priority improvements

3. Fix incrementally:
   - Don't try to fix everything at once
   - Prioritize security and errors
   - Improve code quality over time

4. Use with other tools:
   - Combine with ESLint
   - Use with Error Fixer Agent
   - Integrate with CI/CD

5. Keep agent updated:
   - Add new detection rules
   - Improve fix strategies
   - Update for new patterns
